## Format

Each JSONL line is a single fine-tuning example in OpenAI chat format:

```json
{
  "messages": [
    {
      "role": "system",
      "content": "<CLASSIFICATION_SYSTEM_PROMPT — 5-type taxonomy + Silent Failure Test>"
    },
    {
      "role": "user",
      "content": "CITING CLAIM: Elevated IL-6 causes insulin resistance in skeletal muscle.\n\nRETRIEVED SOURCE PASSAGES:\n[Passage 1, similarity: 0.75]\nOur cross-sectional analysis found serum IL-6 significantly associated with HOMA-IR (r=0.43, p<0.001). Causality cannot be established from this study design.\n\nClassify the distortion type."
    },
    {
      "role": "assistant",
      "content": "{\"distortion_type\": \"causal_overclaim\", \"severity\": 2, \"severity_label\": \"moderate\", \"confidence\": 0.92, \"explanation\": \"Source is cross-sectional and explicitly disclaims causal inference; claim asserts direct causation.\", \"problematic_phrase\": \"\", \"what_source_actually_says\": \"Our cross-sectional analysis found serum IL-6 significantly associated\", \"adequacy_score\": 0.8}"
    }
  ]
}
```

## Training Procedure

```python
from openai import OpenAI
client = OpenAI()

# 1. Upload training file
train_file = client.files.create(
    file=open("data/finetune_train.jsonl", "rb"),
    purpose="fine-tune"
)

# 2. Create fine-tuning job
job = client.fine_tuning.jobs.create(
    training_file=train_file.id,
    model="gpt-4o-mini-2024-07-18",
    hyperparameters={
        "n_epochs": 3,
        "learning_rate_multiplier": 0.1,
    }
)
print(job.id)  # ft:gpt-4o-mini-2024-07-18:org::...

# 3. Monitor
status = client.fine_tuning.jobs.retrieve(job.id)
print(status.status)  # "succeeded"
```

**Hyperparameters:**
| Parameter | Value | Reason |
|-----------|-------|--------|
| `n_epochs` | 3 | Avoids overfitting on 48 examples |
| `learning_rate_multiplier` | 0.1 | Conservative; preserves base model's JSON formatting |
| `model` | `gpt-4o-mini-2024-07-18` | Cost-effective; sufficient for structured classification |

## Evaluation Metrics

After fine-tuning, evaluate using `src/fine_tuning_prep.compute_metrics()`:

| Metric | Target | Description |
|--------|--------|-------------|
| Accuracy | ≥ 85% | Overall label match |
| Precision (macro) | ≥ 85% | Avg precision across all 5 classes |
| Recall (macro) | ≥ 80% | Avg recall across all 5 classes |
| F1 (macro) | ≥ 0.80 | Harmonic mean |
| Cohen's κ | ≥ 0.75 | Agreement corrected for chance |
| Silent failure rate | ≤ 5% | True distortions wrongly labeled "accurate" |

## Why Fine-tuning Improves Performance

1. **Prompt compression (~60% shorter):** The base model requires the full 500-token `CLASSIFICATION_SYSTEM_PROMPT` at every call. A fine-tuned model internalizes the 5-type taxonomy and Silent Failure Test during training — the system prompt can be reduced to a single sentence or omitted entirely. At ~500 tokens/call × 60% reduction × 1,000 calls = **~300,000 tokens saved**.

2. **More consistent JSON output:** The base model occasionally returns free-text preamble before the JSON object, requiring regex repair. Fine-tuning on structured assistant messages teaches strict format adherence from token 1.

3. **Domain calibration:** The base model applies general academic language priors. Fine-tuning on biomedical examples calibrates hedging detection (e.g., recognizing "may suggest" as weaker than "demonstrates") for medical literature.

4. **Reduced hallucination of distortion explanations:** Training on ground-truth explanations anchors the model to cite specific phrases from the source passage rather than generating plausible-sounding but uncited rationale.

## Ethical Note

- Training data is fully synthetic or manually authored — no real patient records, real researcher names, or proprietary paper text.
- The fine-tuned model is intended for citation auditing assistance only, not as an authoritative arbiter of research integrity.
- All outputs should be reviewed by a domain expert before being acted upon.
- The model does not retain or memorize specific paper text between sessions.
